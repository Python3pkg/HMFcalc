Notes on the Parameters used in Calculation

{% endblock %} {% block content %}

The parameters used to calculate the halo mass function with this application can be split into three parts:

  1. Parameters only involved with the production of the transfer function, from which the mass function is derived.
  2. Parameters that are involved only with the calculation of the mass function from the transfer function.
  3. Parameters which affect both of the above explicitly.
The following table summarises the parameters with respect to this categorization.

The transfer function is produced with CAMB. This is done either previously and included by default in this application, uploaded by the user themself, or calculated on the fly. If the user uploads their own transfer function, produced by CAMB either on the web-interface or with the downloadable program, they will have set their own parameters explicitly. In this case, it is important that the cosmological parameters provided explicitly in the HMF application are set to the same values as used for the transfer function. This is the sole responsibility of the user. Note that only parameters in the 'Both' column are necessary to specify accurately in this case, however all parameters are written to ascii files and this may confuse the user at a later date if not specified correctly.

If a default pre-calculated transfer function is used, parameters of that cosmology (specified in the relevant paper) that affect that transfer function in any way are internally set and made non-modifiable so that no confusion arises. Parameters that are relevant to the cosmology and prescribed in the relevant paper, but which do not affect the transfer function, are set to their proper values but are still modifiable.

The difference between the several default transfer functions included in the app lies in the choice of a few cosmological parameters. Otherwise, all other CAMB parameters are equivalent. Each of the sets of parameters are taken from a corresponding paper, linked from the heading.

If a custom transfer function is chosen, but no file is uploaded, then a CAMB run is performed on the fly. This CAMB run is limited in its choice of parameters. Only cosmological parameters may be changed. Otherwise, the CAMB parameters take the same values as outlined above, with the exception of kmax and k-per-log-int which both take the value of 20 for the on-the-fly run. This is to save time.

In summary, the most flexible (but generally inconvenient) way of using this application is to provide your own transfer function produced by CAMB. If so, make sure the cosmological parameters in the form match those used by CAMB. The fastest method is to use a pre-defined transfer function, and these are also quite accurate. However, this is fairly inflexible. A happy medium is to run an on-the-fly transfer function which has 'good enough' accuracy and is very convenient.
